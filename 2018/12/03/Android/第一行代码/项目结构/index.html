第一行代码之Android项目结构

Posted on 2018-12-03 | In Android , 第一行代码

项目结构

  1. .gradle和.idea

    这两个目录下放置的都是Android Studio自动生成的一些文件

  2. app

  3. build

    主要包含一些编译时自动生成的文件

  4. libs

    存放第三方jar包,放在该目录下的jar包都会自动添加到构建路径里去

  5. androidTest

    用来编写AndroidTest测试用例,可以对项目进行一些自动化测试

  6. java

    项目中使用到的所有图片,字符,布局,字符串等资源都要存放在这个目录下.

  7. AndroidManifest.xml

    整个Android项目的配置文件,在程序中定义的四大组件都需要在这个文件里注册,另外还可以在这个文件中给应用程序添加权限声明

  8. test

    用来编写Unit test测试用例

  9. .gitignore

    用于将app模块内的指定的目录或文件排除在版本控制之外

  10. app.iml

    IDEA项目自动生成文件,无须关心

  11. build.gradle

    app模块的gradle构建脚本,该文件会指定很多项目相关的配置

  12. proguard-rules.pro

    用于指定项目代码的混淆规则

  13. build

    包含了一些在编译时自动生成的文件

  14. gradle

    该目录下包含了gradle wrapper的配置文件,自动根据本地缓存的情况决定是否需要联网下载gradle.Android Studio默认没有启用gradle wrapper的方式,如果需要打开,可以通过点击Android Studio导航栏->File->Settings->Build,Excution,Deployment->Gradle,进行配置更改

  15. .gitignore

    将指定的目录或文件排除在版本控制之外

build.gradle

项目全局的gradle构建脚本

  1. gradle.properties

    全局的gradle配置文件

  2. gradlew 和gradlew.bat

    用来在命令行中执行gradle命令,gradlew用于Linux或Mac系统,gradlew.bat用在Windows系统

  3. local.properties

    用于指定本机中的AndroidSDK路径,通常内容都是自动生成的,除非本机的SDK位置发生了变化,则需要修改为新的位置

  4. settings.gradle

    用于指定项目中所有引入的模块,通常情况下模块的引入都是自动完成的
