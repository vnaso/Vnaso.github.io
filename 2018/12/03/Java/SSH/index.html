Hibernate之hbm.xml标签

Posted on 2018-12-03 | In JavaFramework

SSH框架整合

pom.xml

<package>:指定maven项目打包方式

<properties>:用来定义全局变量.用法:${标签名}

<dependencies>:描述当前项目依赖关系

<dependency>:描述当前项目单个依赖关系

<build>:构建项目所需要的信息

<finalName>:指定最后打包的包名

<plugins>:包含插件所需要的信息

纯xml版本

applicationContext.xml内容

Dao部分

  • <bean>的属性已经包含spring需要的所有约束.

  • 为ProductService,ProductDao创建了bean,并注入需要的bean.

Hibernate部分

  • Spring接管SessionFactory的创建,直接作为单例bean使用,简化了创建的操作.bean的创建有两种方式:
    • 通过cfg.xml文件创建
    • 结合properties文件直接给对应property赋值
  • 为HibernateTemplate创建bean,并注入SessionFactory

事务部分

  • 为TransactionManager创建bean,并注入SessionFactory.配置事务管理器
  • 使用<tx:advice>配置事务的通知,属性transaction-manager指定为TransactionManager的bean
    • 子标签<tx:method>用于指定各个事务方法的权限,name用来匹配方法,可以用通配符.其他参照 Spring-notes
  • 使用<aop:config>配置切面
    • <aop:pointcut>用于定义切点
    • <aop:advisor>用于建立切点表达式和事务通知的关联

struts.xml内容

  • <constant name="struts.devMode" value="true"/>开启开发者模式,便于在网页查看错误信息

其他与直接使用struts2时配置相同

hibernate.cfg.xml内容

  • 需要注意:在与spring整合时,<property name="hibernate.current_session_context_class">thread</property>需要删除

spring整合hibernate管理事务后,由Spring的TransactionManager管理事务后, currentSession是绑定到SpringSessionContext的,而不是thread
此时hibernate.current_session_context_class应该是SpringSessionContext,而它又会在使用LocalSessionFactoryBean时自动的设置.所以就不需要你去设置current_session_context_class

参考链接

web.xml配置

  • <filter>标签配置struts2的核心过滤器
  • <listener>配置spring的监听器,在web容器启动时,自动装配applicationContext.xml的配置信息
  • <context-param>自定义applicationContext.xml的位置,不设置则默认位于WEB-INF/目录下

xml+注解版本

总结

  1. 容器应该是单例的
  2. 容器一个应用只有一个,应该存储在应用域中
  3. Struts2的配置文件有加载顺序,遇到相同的配置时,后加载的配置会覆盖先加载的

需要解决的问题

  1. 整合SSH的pom
  2. 集合所有约束的bean.xml
  3. log4j2(书签中)
  4. 使用TransationMangeger时注意导入的class的hibernate后的版本号
  5. maven项目部署到tomcat上 clean->install->tomcat run
  6. web.xml中配置spring的classloader作用及原因
  7. “MySQLNonTransientConnectionException: Public Key Retrieval is not allowed” 重启电脑或软件就能解决 的原因
  8. Struts2的modeldriven模型驱动模式
